

## Retroactive Lyrics Fix for Remaining Orders

### Current State

The previous backfill successfully copied lyrics from leads to orders for all cases where the lead had lyrics. That fixed the majority of affected orders.

**4 orders remain without lyrics**, all delivered with songs but no lyrics text:

| Order | Recipient | Issue |
|-------|-----------|-------|
| 0b4fe283 | Baloo | No linked lead with lyrics |
| e74f80cd | Shanikka | Lead has no lyrics either |
| 458632ec | Katie | Lead has no lyrics either |
| a0b63932 | Walid bezi | Lead has no lyrics either |

These can't be backfilled from leads because the lyrics were never generated in the first place.

### Solution

Trigger the `automation-generate-lyrics` function for each of these 4 orders. This will use their existing story details (special qualities, favorite memory, etc.) to generate lyrics via AI, then store them on the order record. Once generated, the customer song pages will automatically show the lyrics.

### Steps

1. Call `automation-generate-lyrics` for each of the 4 order IDs (via the edge function endpoint)
2. Verify lyrics were generated by checking `automation_lyrics` is populated on each order afterward

### Pitfalls

- **No story data**: If any order is missing `special_qualities` or `favorite_memory`, lyrics generation may fail. All 4 are lead conversions with form data, so this should be fine.
- **Already delivered**: Customers already received their song links. Since the song page dynamically fetches lyrics from the database, the lyrics will appear automatically once generated -- no re-delivery email needed.
- **automation_status**: The lyrics generator will set `automation_status` to `"completed"` automatically.
